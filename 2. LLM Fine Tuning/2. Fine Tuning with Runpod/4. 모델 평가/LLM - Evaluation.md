# sLLM 파인튜닝 모델 평가

---
## 파인튜닝 평가, 왜 필요한가?

파인튜닝(Fine-tuning)은 기존 모델을 특정 작업에 맞게 추가 학습시키는 과정입니다.

> **핵심 질문**: "내가 만든 모델이 정말 잘 되고 있는가?"

평가가 없으면:
- 모델이 더 좋아졌는지 나빠졌는지 알 수 없음
- 과적합(Overfitting) 여부를 확인할 수 없음
- 실제 서비스에 써도 될지 판단 불가

---
## 평가의 두 가지 큰 축
```
평가 방법
├── 자동 평가 (Automatic Evaluation)  → 빠르고 저렴
└── 인간 평가 (Human Evaluation)     → 느리지만 정확
```

---
## 자동 평가 지표 (Automatic Evaluation)

---
### 1. 정량 지표 (숫자로 측정)

| 지표 | 설명 | 주로 쓰이는 곳 |
|------|------|----------------|
| **Perplexity (PPL)** | 모델이 텍스트를 얼마나 "잘 예측"하는지. 낮을수록 좋음 | 언어모델 전반 |
| **BLEU** | 정답과 출력의 단어 일치율 | 번역, 요약 |
| **ROUGE** | 정답과 출력의 겹치는 문장/단어 비율 | 요약 |
| **F1 Score** | 정밀도(Precision)와 재현율(Recall)의 조화평균 | 분류, QA |
| **Accuracy** | 맞춘 비율 | 분류 태스크 |

> **주의**: 자동 지표가 높다고 반드시 좋은 모델이 아닙니다.  
> BLEU가 높아도 문장이 어색할 수 있어요.

---
### 2. LLM-as-a-Judge (AI가 AI를 평가)

강력한 LLM(예: GPT-4, Claude)을 심사자로 써서 출력 품질을 평가하는 방법입니다.
```
[내 파인튜닝 모델 출력] -> [GPT-4 심사] -> 점수 / 피드백
```

**장점**
- 인간 평가에 가까운 품질
- 대규모 평가 자동화 가능

**단점**
- 비용 발생
- 심사 LLM의 편향이 들어올 수 있음

---
## 인간 평가 (Human Evaluation)
자동 평가만으로는 부족할 때 사람이 직접 평가합니다.

---
### 평가 방식

| 방식 | 설명 |
|------|------|
| **절대 평가** | 5점 척도로 출력을 직접 점수 매김 |
| **상대 평가 (A/B)** | 두 모델 출력 중 어느 쪽이 더 좋은지 선택 |
| **Elo Rating** | 여러 모델을 대결시켜 순위 산출 |

---
### 평가 기준 예시

- **정확성**: 사실에 맞는가?
- **유창성**: 자연스러운 문장인가?
- **관련성**: 질문에 적합한 답인가?
- **안전성**: 해로운 내용이 없는가?

# LLM모델 태스크별 평가 전략
파인튜닝 목적에 따라 평가 방법이 달라집니다.

---
## 분류 (Classification)
```
평가 지표: Accuracy, F1 Score, Confusion Matrix
예시: 감성 분석, 스팸 분류
```

## 생성 (Generation)
```
평가 지표: BLEU, ROUGE, BERTScore, Human Eval
예시: 요약, 번역, 글쓰기
```

---
## 질의응답 (QA)
```
평가 지표: Exact Match(EM), F1, Faithfulness
예시: 문서 기반 QA, 챗봇
```

## 지시 따르기 (Instruction Following)
```
평가 지표: LLM-as-a-Judge, MT-Bench 스타일 평가
예시:
```

# 벤치마크 평가
> 공개된 표준 테스트셋으로 모델 성능을 비교합니다.

---
## 주요 벤치마크 (한국어 포함)

| 벤치마크 | 설명 | 언어 |
|----------|------|------|
| **MMLU** | 다분야 지식 이해 (57개 분야) | 영어 |
| **HellaSwag** | 상식 추론 | 영어 |
| **KoBEST** | 한국어 기초 능력 평가 | 한국어 |
| **KLUE** | 한국어 언어 이해 벤치마크 | 한국어 |
| **LogiKAT** | 한국어 논리 추론 | 한국어 |

---
### [벤치마크: lm-evaluation-harness](https://github.com/EleutherAI/lm-evaluation-harness)
- 장점
  - MMLU, HellaSwag 등 60개 이상의 벤치마크가 이미 내장되어 있어서 따로 구현할 필요 없음
  - 커뮤니티가 크고 활발해서 새 벤치마크가 꾸준히 추가됨
  - 재현 가능한 표준화된 평가라 다른 논문/모델과 점수 비교가 쉬움
- 단점
  - 영어 중심 벤치마크가 대부분이라 한국어 평가엔 한계가 있음
  - 큰 모델일수록 평가 시간이 매우 오래 걸림

---
### lm-evaluation-harness vs 다른 평가 모듈

| 항목 | lm-evaluation-harness | Custom Eval (GPT-Judge 등) | HELM | LightEval | 단순 예제 평가 |
|---|---|---|---|---|---|
| 목적 | 표준 성능 비교 | 실사용 품질 검증 | 연구용 종합 분석 | 경량 벤치마크 | 데모 확인 |
| 객관성 | 높음 | 낮음 | 높음 | 높음 | 매우 낮음 |
| 파인튜닝 | 중간 | 매우 높음 | 중간 | 중간 | 매우 낮음 |
| 한국어 | 커스텀 필요 | 매우 쉬움 | 어려움 | 제한적 | 가능 |
| 실무 친화성 | 높음 | 매우 높음 | 낮음 | 중간 | 낮음 |

---
### lm-evaluation-harness + 한국어 특화 평가

| 구분 | lm-evaluation-harness | 한국어 특화 평가 | 이 조합의 의미 |
|---|---|---|---|
| 평가 대상 | 일반 LLM 능력 | 한국어 이해·지시 이행 | 튜닝 목적이 명확해짐 |
| 평가 방식 | 자동 채점 (MCQ 중심) | 정답 기반 + Judge 기반 | 객관성 + 품질 평가 |
| 비교 가능성 | 매우 높음 | 낮음 | 외부 모델과 공정 비교 |
| 튜닝 효과 가시성 | 낮음 | 매우 높음 | 파인튜닝 의미를 설명 가능 |
| 재현성 | 매우 높음 | 중간 | 논문·강의·Hub에 적합 |


---
### lm-eval + 한국어 특화 vs lm-eval + DeepEval

| 평가 관점 | lm-eval + 한국어 특화 | lm-eval + DeepEval |
|---|---|---|
| 평가 대상 | 모델 능력 (언어·지식 중심) | 응답 품질 (행동·추론 중심) |
| 한국어 이해력 | 매우 높음 | 높음 |
| Fine-tuning 효과 | 높음 | 매우 높음 |
| RAG / Tool 사용 적합성 | 매우 낮음 | 매우 높음 |
| 평가 지표 해석 용이성 | 매우 높음 | 중간 |
| 커스터마이징 난이도 | 낮음 | 중간~높음 |
| 외부 비교 가능성 | 높음 | 낮음 |
| 실무 적합성 | 중간 | 매우 높음 |
| 난이도 | 쉬움 | 중간 |
