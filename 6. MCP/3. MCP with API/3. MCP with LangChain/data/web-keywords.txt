HTML (HyperText Markup Language)

정의: HTML은 웹 페이지의 구조와 내용을 정의하기 위해 사용되는 마크업 언어입니다. 웹 브라우저가 텍스트, 이미지, 링크 등의 요소를 어떻게 표시할지 결정합니다.
예시: <h1>제목</h1> 태그는 웹 페이지에 큰 제목을 표시합니다.
연관키워드: 웹 개발, 마크업, 웹 표준

CSS (Cascading Style Sheets)

정의: CSS는 HTML 요소의 스타일과 레이아웃을 정의하는 스타일시트 언어입니다. 색상, 폰트, 간격, 배치 등을 제어하여 웹 페이지의 시각적 표현을 담당합니다.
예시: body { color: blue; }는 웹 페이지의 텍스트 색상을 파란색으로 설정합니다.
연관키워드: 웹 디자인, 스타일링, 레이아웃

JavaScript

정의: JavaScript는 웹 페이지에 동적인 기능을 추가하기 위한 프로그래밍 언어입니다. 사용자 상호작용, 데이터 처리, DOM 조작 등을 수행할 수 있습니다.
예시: 버튼을 클릭하면 알림창이 나타나도록 하는 기능은 JavaScript로 구현할 수 있습니다.
연관키워드: 웹 개발, 프론트엔드, 동적 웹페이지

React

정의: React는 사용자 인터페이스를 구축하기 위한 JavaScript 라이브러리입니다. 컴포넌트 기반 아키텍처를 사용하여 재사용 가능하고 유지보수하기 쉬운 웹 애플리케이션을 만들 수 있습니다.
예시: React를 사용하여 재사용 가능한 버튼 컴포넌트를 만들고 여러 페이지에서 사용할 수 있습니다.
연관키워드: 프론트엔드 프레임워크, 컴포넌트, 가상 DOM

Vue.js

정의: Vue.js는 사용자 인터페이스를 구축하기 위한 프로그레시브 JavaScript 프레임워크입니다. 점진적으로 채택할 수 있으며, 작은 프로젝트부터 대규모 애플리케이션까지 확장 가능합니다.
예시: Vue.js를 사용하여 양방향 데이터 바인딩을 구현하여 사용자 입력이 즉시 화면에 반영되도록 할 수 있습니다.
연관키워드: 프론트엔드 프레임워크, 반응형, 컴포넌트

Angular

정의: Angular는 단일 페이지 애플리케이션(SPA)을 구축하기 위한 TypeScript 기반 웹 애플리케이션 프레임워크입니다. 엔터프라이즈급 애플리케이션 개발에 적합합니다.
예시: Angular를 사용하여 복잡한 대시보드 애플리케이션을 구축하고 라우팅, 상태 관리 등을 처리할 수 있습니다.
연관키워드: 프론트엔드 프레임워크, TypeScript, SPA

Node.js

정의: Node.js는 서버 측에서 JavaScript를 실행할 수 있게 해주는 런타임 환경입니다. 비동기 이벤트 기반 아키텍처를 사용하여 확장 가능한 네트워크 애플리케이션을 구축할 수 있습니다.
예시: Node.js를 사용하여 RESTful API 서버를 구축하고 데이터베이스와 상호작용할 수 있습니다.
연관키워드: 서버 사이드, JavaScript, 비동기 프로그래밍

Express

정의: Express는 Node.js를 위한 최소한의 유연한 웹 애플리케이션 프레임워크입니다. 웹 애플리케이션과 API를 빠르고 쉽게 구축할 수 있도록 도와줍니다.
예시: Express를 사용하여 라우팅, 미들웨어, 템플릿 엔진 등을 활용한 웹 서버를 구축할 수 있습니다.
연관키워드: Node.js, 웹 프레임워크, API 개발

REST API

정의: REST API는 웹 서비스 아키텍처 스타일로, HTTP 메서드를 사용하여 리소스에 접근하고 조작하는 방식입니다. 표준화된 인터페이스를 제공하여 클라이언트와 서버 간의 통신을 용이하게 합니다.
예시: GET /api/users는 사용자 목록을 조회하고, POST /api/users는 새 사용자를 생성합니다.
연관키워드: 웹 API, HTTP, 서버 통신

GraphQL

정의: GraphQL은 API를 위한 쿼리 언어이자 런타임입니다. 클라이언트가 필요한 데이터를 정확히 요청할 수 있도록 하여 효율적인 데이터 페칭을 가능하게 합니다.
예시: GraphQL 쿼리를 사용하여 사용자 정보와 그들의 게시물을 한 번의 요청으로 가져올 수 있습니다.
연관키워드: API, 데이터 쿼리, 효율적인 데이터 페칭

DOM (Document Object Model)

정의: DOM은 HTML 문서의 구조를 객체로 표현한 것으로, 프로그래밍 언어가 문서의 구조, 스타일, 내용에 접근하고 조작할 수 있게 해줍니다.
예시: JavaScript를 사용하여 DOM을 조작하여 버튼 클릭 시 텍스트를 변경할 수 있습니다.
연관키워드: 웹 개발, JavaScript, HTML

AJAX (Asynchronous JavaScript and XML)

정의: AJAX는 웹 페이지를 새로고침하지 않고도 서버와 비동기적으로 데이터를 교환하는 기술입니다. 사용자 경험을 향상시키고 더 반응형 웹 애플리케이션을 만들 수 있게 해줍니다.
예시: 검색어를 입력하면 페이지를 새로고침하지 않고 자동완성 결과가 나타나는 기능은 AJAX로 구현됩니다.
연관키워드: 비동기 통신, 웹 개발, 사용자 경험

WebSocket

정의: WebSocket은 단일 TCP 연결을 통해 양방향 통신을 제공하는 프로토콜입니다. 실시간 데이터 교환이 필요한 애플리케이션에 적합합니다.
예시: 채팅 애플리케이션에서 메시지를 실시간으로 주고받는 기능은 WebSocket을 사용하여 구현할 수 있습니다.
연관키워드: 실시간 통신, 양방향 통신, 네트워크 프로토콜

HTTP (HyperText Transfer Protocol)

정의: HTTP는 웹에서 데이터를 전송하기 위한 프로토콜입니다. 클라이언트와 서버 간의 요청과 응답을 처리하여 웹 페이지와 리소스를 전송합니다.
예시: 브라우저가 웹 페이지를 요청하면 서버는 HTTP 프로토콜을 통해 HTML 문서를 응답으로 전송합니다.
연관키워드: 웹 프로토콜, 네트워크 통신, 인터넷

HTTPS (HyperText Transfer Protocol Secure)

정의: HTTPS는 HTTP의 보안 버전으로, SSL/TLS 암호화를 사용하여 데이터 전송의 보안을 보장합니다. 민감한 정보를 안전하게 전송할 수 있게 해줍니다.
예시: 온라인 쇼핑몰에서 결제 정보를 입력할 때 HTTPS를 사용하여 데이터가 암호화되어 전송됩니다.
연관키워드: 보안, 암호화, SSL/TLS

Cookie

정의: 쿠키는 웹 서버가 사용자의 브라우저에 저장하는 작은 데이터 조각입니다. 사용자 세션 관리, 개인화 설정 저장 등에 사용됩니다.
예시: 웹사이트에 로그인하면 쿠키에 인증 정보가 저장되어 다음 방문 시 자동으로 로그인 상태를 유지할 수 있습니다.
연관키워드: 세션 관리, 웹 저장소, 사용자 추적

Session

정의: 세션은 서버에 저장되는 사용자 상태 정보입니다. 사용자가 웹사이트를 방문하는 동안 유지되며, 로그인 상태나 장바구니 정보 등을 관리하는 데 사용됩니다.
예시: 온라인 쇼핑몰에서 장바구니에 담은 상품 정보는 세션에 저장되어 사용자가 다른 페이지로 이동해도 유지됩니다.
연관키워드: 상태 관리, 서버 저장소, 사용자 인증

LocalStorage

정의: LocalStorage는 웹 브라우저에 데이터를 영구적으로 저장할 수 있는 웹 스토리지 API입니다. 쿠키보다 더 많은 데이터를 저장할 수 있으며, 서버로 전송되지 않습니다.
예시: 사용자의 테마 설정(다크 모드/라이트 모드)을 LocalStorage에 저장하여 다음 방문 시에도 유지할 수 있습니다.
연관키워드: 웹 저장소, 클라이언트 저장소, 브라우저 API

SessionStorage

정의: SessionStorage는 웹 브라우저에 데이터를 세션 동안만 저장하는 웹 스토리지 API입니다. 브라우저 탭이 닫히면 데이터가 삭제됩니다.
예시: 폼 작성 중 임시로 입력한 데이터를 SessionStorage에 저장하여 실수로 페이지를 새로고침해도 데이터가 유지되도록 할 수 있습니다.
연관키워드: 웹 저장소, 세션 저장소, 브라우저 API

CORS (Cross-Origin Resource Sharing)

정의: CORS는 다른 도메인에서 리소스에 접근할 수 있도록 허용하는 메커니즘입니다. 웹 보안 정책을 준수하면서도 다른 출처의 리소스를 안전하게 사용할 수 있게 해줍니다.
예시: 프론트엔드 애플리케이션이 다른 도메인의 API를 호출할 때 CORS 설정이 필요합니다.
연관키워드: 웹 보안, API 통신, 도메인 간 통신

JWT (JSON Web Token)

정의: JWT는 두 당사자 간에 정보를 안전하게 전송하기 위한 컴팩트하고 자가 수용적인 방식입니다. 주로 인증 및 권한 부여에 사용됩니다.
예시: 사용자가 로그인하면 서버가 JWT를 발급하고, 이후 요청에서 이 토큰을 사용하여 사용자 인증을 확인합니다.
연관키워드: 인증, 보안, 토큰 기반 인증

OAuth

정의: OAuth는 타사 애플리케이션에 사용자 리소스에 대한 제한된 접근 권한을 부여하는 인증 프로토콜입니다. 사용자 비밀번호를 공유하지 않고도 안전하게 인증할 수 있습니다.
예시: "Google로 로그인" 버튼을 클릭하면 OAuth를 통해 Google 계정으로 인증할 수 있습니다.
연관키워드: 인증, 권한 부여, 소셜 로그인

Responsive Design

정의: 반응형 디자인은 다양한 화면 크기와 디바이스에 맞게 웹 페이지의 레이아웃이 자동으로 조정되는 디자인 접근 방식입니다. 모바일, 태블릿, 데스크톱에서 모두 최적의 사용자 경험을 제공합니다.
예시: 모바일에서는 세로로 배치된 메뉴가 데스크톱에서는 가로로 배치되는 웹사이트는 반응형 디자인을 사용합니다.
연관키워드: 웹 디자인, 모바일 최적화, 사용자 경험

PWA (Progressive Web App)

정의: PWA는 웹 기술을 사용하여 네이티브 앱과 유사한 경험을 제공하는 웹 애플리케이션입니다. 오프라인 작동, 푸시 알림, 홈 화면 설치 등의 기능을 제공합니다.
예시: PWA로 구축된 뉴스 앱은 오프라인에서도 이전에 로드한 기사를 읽을 수 있고, 새 기사가 업데이트되면 푸시 알림을 받을 수 있습니다.
연관키워드: 웹 앱, 오프라인 기능, 모바일 웹

TypeScript

정의: TypeScript는 JavaScript에 정적 타입을 추가한 프로그래밍 언어입니다. 컴파일 시 타입 체크를 수행하여 오류를 사전에 발견하고 코드의 안정성을 높입니다.
예시: TypeScript를 사용하여 변수의 타입을 명시적으로 선언하면, 잘못된 타입의 값을 할당하려고 할 때 컴파일 오류가 발생합니다.
연관키워드: JavaScript, 타입 안정성, 웹 개발

Webpack

정의: Webpack은 JavaScript 모듈 번들러로, 여러 파일과 의존성을 하나 이상의 번들로 묶어주는 도구입니다. 최적화, 코드 분할, 자산 관리 등의 기능을 제공합니다.
예시: Webpack을 사용하여 여러 JavaScript 파일을 하나로 묶고, CSS와 이미지를 최적화하여 웹 애플리케이션을 배포할 수 있습니다.
연관키워드: 모듈 번들러, 빌드 도구, 프론트엔드 개발

Babel

정의: Babel은 최신 JavaScript 코드를 이전 버전의 JavaScript로 변환하는 트랜스파일러입니다. 최신 문법을 사용하면서도 오래된 브라우저에서도 동작하도록 해줍니다.
예시: ES6의 화살표 함수나 async/await 문법을 사용한 코드를 Babel이 ES5 코드로 변환하여 구형 브라우저에서도 실행할 수 있게 합니다.
연관키워드: JavaScript, 트랜스파일러, 브라우저 호환성

Git

정의: Git은 분산 버전 관리 시스템으로, 소스 코드의 변경 이력을 추적하고 관리할 수 있게 해줍니다. 협업 개발과 코드 버전 관리를 위한 필수 도구입니다.
예시: 개발자가 코드를 수정하면 Git을 사용하여 변경 사항을 커밋하고, 팀원들과 공유할 수 있습니다.
연관키워드: 버전 관리, 협업, 소스 코드 관리

GitHub

정의: GitHub는 Git 저장소를 호스팅하는 웹 기반 플랫폼입니다. 코드 저장, 협업, 이슈 추적, 코드 리뷰 등의 기능을 제공합니다.
예시: 개발자들이 GitHub에 프로젝트를 업로드하고, 다른 개발자들이 코드를 복제하여 기여할 수 있습니다.
연관키워드: Git, 코드 호스팅, 협업 플랫폼

CI/CD (Continuous Integration/Continuous Deployment)

정의: CI/CD는 코드 변경 사항을 자동으로 통합, 테스트, 배포하는 개발 프로세스입니다. 개발 속도를 높이고 배포 품질을 향상시킵니다.
예시: 코드를 GitHub에 푸시하면 자동으로 테스트가 실행되고, 테스트가 통과하면 자동으로 프로덕션 환경에 배포됩니다.
연관키워드: 자동화, DevOps, 소프트웨어 배포

Docker

정의: Docker는 애플리케이션을 컨테이너로 패키징하여 배포할 수 있게 해주는 플랫폼입니다. 환경 독립성을 보장하여 "내 컴퓨터에서는 작동했는데" 문제를 해결합니다.
예시: Docker를 사용하여 개발 환경과 프로덕션 환경을 동일하게 구성하여 배포 시 발생할 수 있는 문제를 방지할 수 있습니다.
연관키워드: 컨테이너, 가상화, 배포

Kubernetes

정의: Kubernetes는 컨테이너 오케스트레이션 플랫폼으로, 대규모 컨테이너 애플리케이션의 배포, 확장, 관리를 자동화합니다.
예시: Kubernetes를 사용하여 웹 애플리케이션의 인스턴스를 자동으로 확장하고, 장애가 발생한 컨테이너를 자동으로 재시작할 수 있습니다.
연관키워드: 컨테이너 오케스트레이션, 클라우드, 확장성

AWS (Amazon Web Services)

정의: AWS는 아마존에서 제공하는 클라우드 컴퓨팅 플랫폼입니다. 서버, 스토리지, 데이터베이스, 네트워킹 등 다양한 클라우드 서비스를 제공합니다.
예시: 웹 애플리케이션을 AWS의 EC2 인스턴스에 배포하고, S3에 정적 파일을 저장할 수 있습니다.
연관키워드: 클라우드 컴퓨팅, 인프라, 서버 호스팅

SEO (Search Engine Optimization)

정의: SEO는 검색 엔진에서 웹사이트의 순위를 높이기 위한 최적화 기법입니다. 검색 결과에서 더 높은 순위를 얻어 자연스러운 트래픽을 증가시킵니다.
예시: 웹 페이지의 메타 태그를 최적화하고, 의미 있는 URL 구조를 사용하여 검색 엔진이 콘텐츠를 더 잘 이해하도록 할 수 있습니다.
연관키워드: 검색 엔진, 웹 마케팅, 콘텐츠 최적화

Accessibility (접근성)

정의: 웹 접근성은 장애를 가진 사용자도 웹사이트를 사용할 수 있도록 보장하는 것입니다. 스크린 리더 지원, 키보드 네비게이션, 색상 대비 등을 고려합니다.
예시: 이미지에 alt 텍스트를 추가하고, 키보드만으로도 모든 기능을 사용할 수 있도록 웹사이트를 설계하면 접근성이 향상됩니다.
연관키워드: 웹 표준, 사용자 경험, 포용성
